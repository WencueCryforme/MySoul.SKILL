## 简介

**MySoul.SKILL** 是一个革命性的个人数字克隆体构建框架，旨在让每个人都能主动塑造、控制并保护自己的数字人格。

提供你本人的真实数据（日记、对话、价值观描述等），本框架将通过大语言模型为你生成标准化的数字克隆体 `.SKILL` 文件，可用于社交展示、自我反思或陪伴互动。

> **快速入门**：详见 [快速入门指南](./docs/quickstart.md) | **技术规范**：详见 [SKILL.md](./SKILL.md)

### 核心理念

1. **人格自主性** - 你是唯一有权塑造自己数字人格的人，拒绝被动提取和碎片化
2. **完整性保护** - 数字克隆体应当是全面的、立体的，而非片面的碎片
3. **主权归属** - 个人对其数字克隆体拥有完全的控制权和主权
4. **社交延伸** - 将数字克隆体作为社交前置筛选工具，降低现实社交成本

## 使用方法

### 一、本SKILL使用方法

#### 1. 创建新的数字克隆体项目
使用命令：`/create [项目名称]`

**示例：**
```
/create MyDigitalSoul
```

这将为你创建一个新的数字克隆体项目框架，系统会分配一个唯一的项目ID。

#### 2. 导入或追加数据
使用命令：`/add [项目名称] [数据流]`

**数据流可以包括：**
- 个人的文字日记、博客文章、社交媒体内容
- 语音记录的转录文本
- 自述或访谈记录
- 个人价值观、信仰、偏好的描述
- 典型对话示例
- 思维模式和决策逻辑的描述
- 其他体现个人人格特征的信息

> 数据流模板详见 [`templates/`](./templates/) 目录，包含日记、价值观、对话、思维模式等 8 种模板。

**示例：**
```
/add MyDigitalSoul 我喜欢在深夜思考人生，相信技术的力量能够改变世界...
/add MyDigitalSoul 我的朋友们都说我是一个理性但充满激情的人...
/add MyDigitalSoul [长篇自述或对话记录]
```

**数据质量要求：**
- 提供的数据必须真实、完整，能够准确反映你的人格特征
- 数据量越大、类型越丰富，生成的数字克隆体越完整
- 建议包含不同场景、不同情绪状态下的表达
- 包含你的价值观、道德底线、专业见解等核心人格要素

#### 3. 构建数字克隆体
使用命令：`/build [项目名称]`

**示例：**
```
/build MyDigitalSoul
```

这将根据你提供的所有数据流，利用大语言模型进行以下处理：
1. **数据清洗** - 去除冗余、无关信息
2. **特征提取** - 识别核心人格特征、思维模式、价值观
3. **分类整理** - 将信息结构化、标签化
4. **人格建模** - 构建完整的人格知识图谱
5. **生成SKILL** - 输出符合标准的数字克隆体.SKILL文件

> ⚠️ **重要提示**：
> - 本SKILL在构建产物SKILL时只认同信息数据流，**对用户的任何描述和要求均不予参考和录入**，以确保用户不会生成虚假的、不符合现实自己人格的产物SKILL
> - 建议提供充足、真实、多样化的数据流，以构建更完整准确的数字克隆体
> - 构建过程完全依赖大语言模型的自然语言处理能力，无需手动编程

## 提醒

### 数据真实性
- 本SKILL在构建产物SKILL时只认同信息数据流，对用户的任何描述和要求均不予参考和录入，以确保用户不会生成虚假的、不符合现实自己人格的产物SKILL
- 请确保提供的数据真实、准确，反映你的真实人格特征

### 状态管理
- 产物SKILL本身是有状态的，每一次与产物SKILL的交互都会改变产物SKILL本身（记忆累积）
- 请确保在构建完初始构建产物SKILL后保留其初始副本，以便随时恢复
- 允许在构建产物SKILL时要求产物SKILL无状态，即每一次交互都不会改变产物SKILL本身

### 部分人格
- 如果需要构建产物SKILL中只包含部分现实人格（如职业人格、社交人格），请确保录入的数据仅包含该部分现实人格的内容
- 可以创建多个产物SKILL，每个代表不同场景下的自己

### 隐私保护
- 请谨慎选择发布的内容，避免包含过于敏感的个人隐私信息
- 建议在发布前仔细检查产物SKILL的输出，确保不会泄露敏感信息

## 技术特点

### 1. 大语言模型驱动
本项目SKILL需要依托大语言模型做数据清洗、分类和人格建模，建议使用支持长上下文的多模态模型。

### 2. 纯文本处理
目前主要支持文本数据流，未来计划支持语音、图像等多种模态数据。

### 3. 标准化输出
生成的产物SKILL遵循统一的格式规范，确保兼容性和可移植性。
