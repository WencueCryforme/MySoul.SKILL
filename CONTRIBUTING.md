# 贡献指南

感谢你对 MySoul.SKILL 项目的关注！本指南说明如何参与贡献。

---

## 如何贡献

### 报告问题
1. 在 [GitHub Issues](https://github.com/WencueCryforme/MySoul.SKILL/issues) 提交问题
2. 使用清晰的标题描述问题
3. 提供复现步骤和环境信息
4. 附上相关截图或日志

### 提交功能建议
1. 在 Issues 中提交功能建议
2. 说明使用场景和预期效果
3. 如果可能，提供设计草案

### 提交代码
1. Fork 本仓库
2. 创建功能分支：`git checkout -b feature/your-feature`
3. 提交更改：`git commit -m "Add your feature"`
4. 推送分支：`git push origin feature/your-feature`
5. 创建 Pull Request

---

## 开发规范

### 文档规范
- 使用中文编写主要文档
- 提供英文翻译（可选）
- 保持简洁精准，节省字数
- 使用 Markdown 格式

### SKILL 规范
- 遵循 MySoul.SKILL 标准格式
- 新增模块需更新 SKILL.md
- 新增维度需更新 extraction-rules.md
- 新增命令需更新 .claude/commands/

### 提交规范
- 使用清晰的提交信息
- 每个提交只做一件事
- 提交前检查文件完整性

---

## Pull Request 流程

### PR 要求
- 标题清晰描述变更内容
- 说明变更的原因和影响
- 关联相关 Issue
- 通过所有检查

### 审查流程
1. 维护者审查代码
2. 可能要求修改
3. 审查通过后合并
4. 更新 CHANGELOG

---

## 行为准则

### 尊重他人
- 尊重不同观点和意见
- 使用友善和专业的语言
- 不进行人身攻击

### 真实贡献
- 不提交虚假信息
- 不抄袭他人作品
- 不恶意破坏项目
