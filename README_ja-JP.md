## 紹介

**MySoul.SKILL** は、革命的な個人のデジタルクローン構築フレームワークであり、すべての人が自分のデジタル人格を能動的に形成、制御、保護できるようにすることを目指しています。

あなた自身の本物のデータ（日記、会話、価値観の説明など）を提供すると、このフレームワークは大規模言語モデルを通じて標準化されたデジタルクローン `.SKILL` ファイルを生成します。ソーシャル展示、自己反省、同行者として使用できます。

> **クイックスタート**: [クイックスタートガイド](./docs/quickstart.md) | **技術仕様**: [SKILL.md](./SKILL.md)

### 中核理念

1. **人格自律性** - あなたは自分のデジタル人格を形成する権利を持つ唯一の存在であり、受動的な抽出と断片化を拒否します
2. **完全性保護** - デジタルクローンは全面的で立体的であるべきであり、断片化的であってはなりません
3. **主権帰属** - 個人は自分のデジタルクローンに対して完全な制御権と主権を持ちます
4. **社会的拡張** - デジタルクローンを社会的な事前スクリーニングツールとして使用し、現実の社会的コストを削減します

```bash
git clone https://github.com/WencueCryforme/MySoul.SKILL.git
cd MySoul.SKILL
```

## 使用方法

### 1. MySoul.SKILL の使用方法

#### 新しいデジタルクローンプロジェクトを作成
コマンド：`/create [プロジェクト名]`

**例：**
```
/create MyDigitalSoul
```

これにより、あなたのために新しいデジタルクローンプロジェクトフレームワークが作成され、システムが一意のプロジェクトIDを割り当てます。

#### データのインポートまたは追加
コマンド：`/add [プロジェクト名] [データストリーム]`

**データストリームには以下が含まれます：**
- 個人のテキスト日記、ブログ記事、ソーシャルメディアコンテンツ
- 音声録音の転写テキスト
- 自己紹介またはインタビュー記録
- 個人の価値観、信念、好みの説明
- 典型的な会話例
- 思考パターンと意思決定ロジックの説明
- 個人の人格特徴を反映するその他の情報

> データストリームのテンプレートは [`templates/`](./templates/) ディレクトリにあり、日記、価値観、会話、思考パターンなど8種類のテンプレートが含まれています。

**例：**
```
/add MyDigitalSoul 私は夜遅くに人生について考え、テクノロジーの力が世界を変えることができると信じています...
/add MyDigitalSoul 私の友人たちは皆、私は理性的で情熱的な人だと言います...
/add MyDigitalSoul [長い自己紹介または会話記録]
```

**データ品質要件：**
- 提供されたデータは真実で完全であり、あなたの人格特徴を正確に反映している必要があります
- データ量が大きく、タイプが多様であるほど、生成されるデジタルクローンは完全になります
- 異なるシナリオ、異なる感情的状態での表現を含めることをお勧めします
- あなたの価値観、道徳的底线、専門的見解などの核心的人格要素を含めます

#### デジタルクローンを構築
コマンド：`/build [プロジェクト名]`

**例：**
```
/build MyDigitalSoul
```

これにより、提供されたすべてのデータストリームに基づいて、大規模言語モデルを使用して以下の処理が行われます：
1. **データクリーニング** - 冗長で無関係な情報を削除
2. **特徴抽出** - 核心的人格特徴、思考パターン、価値観を識別
3. **分類と整理** - 情報を構造化し、タグ付け
4. **人格モデリング** - 完全な人格知識グラフを構築
5. **SKILLの生成** - 標準化されたデジタルクローン.SKILLファイルを出力

> ⚠️ **重要な注意事項：**
> - 製品SKILLを構築する際、このSKILLは情報データストリームのみを認識し、ユーザーの記述や要求を参照・記録しないため、ユーザーが現実の自分の人格と一致しない偽の製品SKILLを生成するのを防ぎます。
> - より完全で正確なデジタルクローンを構築するために、十分で真実、多様なデータストリームを提供することをお勧めします。
> - 構築プロセスは完全に大規模言語モデルの自然言語処理機能に依存しており、手動プログラミングは必要ありません。

## 重要な注意事項

### データの真実性
- 製品SKILLを構築する際、このSKILLは情報データストリームのみを認識し、ユーザーの記述や要求を参照・記録しないため、ユーザーが現実の自分の人格と一致しない偽の製品SKILLを生成するのを防ぎます。
- 提供されたデータが真実で正確であり、あなたの真実の人格特徴を反映していることを確認してください。

### 状態管理
- 製品SKILL自体はステートフルであり、製品SKILLとのすべての対話が製品SKILL自体を変更します（累積的記憶）。
- 初期製品SKILLを構築した後、初期コピーを保持して、いつでも復元できるようにすることをお勧めします。
- 構築時に製品SKILLをステートレスにすることを要求できます。つまり、各対話が製品SKILL自体を変更しないことを意味します。

### 部分的人格
- 部分的な現実的人格（職業的人格、社会的的人格など）のみを含む製品SKILLを構築する必要がある場合、インポートされたデータがその部分的な現実的人格のコンテンツのみを含むことを確認してください。
- 複数の製品SKILLを作成でき、それぞれがあなたの異なる側面を表します。

### プライバシー保護
- 公開するものを慎重に選択し、過度に敏感な個人のプライバシー情報を含めるのを避けてください。
- 公開する前に製品SKILLの出力を慎重に確認し、機密情報が漏洩していないことを確認することをお勧めします。

## 技術的特徴

### 1. LLM駆動
このプロジェクトSKILLは、データクリーニング、分類、人格モデリングのために大規模言語モデルに依存しています。長いコンテキストのマルチモーダルモデルを使用することをお勧めします。

### 2. テキストのみの処理
現在、主にテキストデータストリームをサポートしています。将来の計画には、音声や画像などの複数のモダリティのサポートが含まれます。

### 3. 標準化された出力
生成された製品SKILLは統一されたフォーマット仕様に従い、互換性と移植性を確保します。
