## Introduction

**MySoul.SKILL** is a revolutionary framework for building personal digital clones. It aims to enable everyone to actively shape, control, and protect their own digital personality.

Provide your authentic personal data (diaries, conversations, value descriptions, etc.), and this framework will generate a standardized digital clone `.SKILL` file through large language models, suitable for social showcase, self-reflection, or companionship.

> **Quick Start**: See [Quick Start Guide](./docs/quickstart.md) | **Technical Spec**: See [SKILL.md](./SKILL.md)

### Core Philosophy

1. **Personality Autonomy** - You are the only one with the right to shape your own digital personality; refuse passive extraction and fragmentation.
2. **Integrity Protection** - Digital clones should be comprehensive and multidimensional, not fragmented fragments.
3. **Sovereignty Belonging** - Individuals have complete control and sovereignty over their digital clones.
4. **Social Extension** - Use digital clones as a pre-social screening tool to reduce real-world social costs.

## Usage

### 1. Using MySoul.SKILL

#### Create a New Digital Clone Project
Use command: `/create [project_name]`

**Example:**
```
/create MyDigitalSoul
```

This will create a new digital clone project framework for you, and the system will assign a unique project ID.

#### Import or Append Data
Use command: `/add [project_name] [data_stream]`

**Data streams can include:**
- Personal diaries, blog posts, social media content
- Transcripts of voice recordings
- Self-descriptions or interview records
- Descriptions of personal values, beliefs, preferences
- Typical conversation examples
- Descriptions of thinking patterns and decision logic
- Other information reflecting personal personality traits

> Data stream templates are available in the [`templates/`](./templates/) directory, including 8 types: diary, values, conversation, thinking patterns, etc.

**Example:**
```
/add MyDigitalSoul I like thinking about life late at night, and I believe in the power of technology to change the world...
/add MyDigitalSoul My friends all say I am a rational but passionate person...
/add MyDigitalSoul [Long self-description or conversation records]
```

**Data Quality Requirements:**
- The data provided must be authentic and complete, accurately reflecting your personality traits
- The larger and more diverse the data volume, the more complete and accurate the generated digital clone will be
- It is recommended to include expressions in different scenarios and emotional states
- Include core personality elements such as values, moral bottom lines, and professional insights

#### Build Digital Clone
Use command: `/build [project_name]`

**Example:**
```
/build MyDigitalSoul
```

This will use all the data streams you provided and leverage large language models to perform the following processing:
1. **Data Cleaning** - Remove redundant and irrelevant information
2. **Feature Extraction** - Identify core personality traits, thinking patterns, and values
3. **Classification and Organization** - Structure and tag information
4. **Personality Modeling** - Build a complete personality knowledge graph
5. **Generate SKILL** - Output a standardized digital clone .SKILL file

> ⚠️ **Important Notes:**
> - When building a product SKILL, this SKILL only recognizes information data streams and does not refer to or record any descriptions or requirements from the user, ensuring that users do not generate fake product SKILLs that do not match their real personality.
> - It is recommended to provide sufficient, authentic, and diverse data streams to build a more complete and accurate digital clone.
> - The building process relies entirely on the natural language processing capabilities of large language models, requiring no manual programming.

## Important Notes

### Data Authenticity
- When building a product SKILL, this SKILL only recognizes information data streams and does not refer to or record any descriptions or requirements from the user, ensuring that users do not generate fake product SKILLs that do not match their real personality.
- Please ensure the data provided is authentic and accurate, reflecting your true personality traits.

### State Management
- The product SKILL itself is stateful, and every interaction with the product SKILL will change the product SKILL itself (cumulative memory).
- Please ensure that after building the initial product SKILL, you keep an initial copy of it so you can restore it at any time.
- You are allowed to require the product SKILL to be stateless when building it, meaning that each interaction will not change the product SKILL itself.

### Partial Personality
- If you need to build a product SKILL that contains only partial real personality (such as professional personality, social personality), please ensure that the imported data only contains the content of that partial real personality.
- You can create multiple product SKILLs, each representing different aspects of yourself.

### Privacy Protection
- Please choose what you publish carefully to avoid including overly sensitive personal privacy information.
- It is recommended to carefully check the output of the product SKILL before publishing to ensure it does not leak sensitive information.
