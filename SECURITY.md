# 安全策略

---

## 报告漏洞

如果你发现安全漏洞，请通过以下方式报告：

1. **不要**在公开 Issue 中报告
2. 通过 GitHub 私信联系维护者
3. 或发送邮件至安全联系邮箱

---

## 漏洞处理流程

1. 收到报告后 48 小时内确认
2. 评估漏洞严重程度
3. 开发修复方案
4. 发布安全更新
5. 公开披露（修复后）

---

## 安全最佳实践

### 用户数据保护
- 不收集用户个人身份信息
- 数据传输使用加密
- 用户可随时删除数据

### 产物 SKILL 安全
- 产物 SKILL 不包含原始数据
- 人格特征经过脱敏处理
- 遵守 PLOSL 协议

### 构建安全
- 构建过程本地执行
- 不上传用户数据到外部服务器
- 构建产物本地存储

---

## 安全更新

安全更新将通过以下方式发布：
- GitHub Security Advisories
- CHANGELOG.md
- README.md 公告
