# 人格一致性检查清单

构建产物 SKILL 后，使用此清单进行一致性校验。

---

## 检查项

### 1. 内部一致性（各维度之间不矛盾）

- [ ] 核心价值观与生活态度一致
  - 例：声称"重视家庭"但工作观显示"工作至上" → 矛盾
- [ ] 思维模式与表达风格一致
  - 例：声称"理性分析型"但表达风格偏"情绪化" → 矛盾
- [ ] 情感特征与人际互动指南一致
  - 例：声称"内向"但社交偏好为"主动社交" → 需要解释
- [ ] 知识体系与专业领域一致
  - 例：声称"技术专家"但知识深度为"入门" → 矛盾

### 2. 数据一致性（特征与数据一致）

- [ ] 每个人格特征都有数据支撑
- [ ] 没有从单条数据过度推导
- [ ] 表达风格与实际数据中的表达一致
- [ ] 没有编造或臆测的特征

### 3. 完整性检查

- [ ] 核心价值观 - 已填写
- [ ] 思维模式 - 已填写
- [ ] 表达风格 - 已填写
- [ ] 知识体系 - 已填写
- [ ] 人际关系 - 已填写
- [ ] 情感特征 - 已填写
- [ ] 生活态度 - 已填写
- [ ] 开场白 - 已填写
- [ ] 示例对话 - 已填写（2-3 组）
- [ ] 对话指南 - 已填写
- [ ] 边界与限制 - 已填写
- [ ] 记忆机制 - 已填写
- [ ] 元信息 - 已填写

### 4. 可用性检查

- [ ] 开场白自然、体现人格风格
- [ ] 示例对话覆盖多种场景
- [ ] 边界列表完整（隐私、正式意见、敏感话题）
- [ ] 记忆机制明确（有状态/无状态）
- [ ] 语言风格与创作者一致

### 5. 协议合规

- [ ] 包含身份声明
- [ ] 引用 PLOSL License
- [ ] 不包含用户原始数据的直接引用

## 评分标准

| 维度 | 权重 | 评分方法 |
|------|------|----------|
| 内部一致性 | 25% | 矛盾项数：0=25分，1=20分，2=15分，3+=0分 |
| 数据一致性 | 25% | 无支撑特征数：0=25分，1-2=20分，3-5=15分，5+=0分 |
| 完整性 | 25% | 已填写模块数/总模块数 × 25 |
| 可用性 | 25% | 主观评估：优秀=25，良好=20，基本=15，不足=0 |

总分 100 分：
- 90+：优秀，可直接使用
- 70-89：良好，建议微调
- 50-69：基本，需要修改
- <50：建议补充数据后重新构建

## 常见问题

**Q: 发现矛盾怎么办？**
A: 分析哪个特征有更多数据支撑，保留数据支撑更强的特征，标注矛盾点。

**Q: 某维度数据不足怎么办？**
A: 标注"数据不足，待补充"，不要编造。可以建议用户补充该维度的数据。

**Q: 表达风格太泛怎么办？**
A: 从数据中提取具体的表达习惯，如"常用'嗯'开头"、"喜欢用比喻"等。
