# 人格特征提取规则

本文档定义 `/build` 命令在特征提取阶段的具体执行规则。

---

## 提取原则

1. **数据驱动** - 每个特征必须有至少一条数据支撑
2. **推导而非臆造** - 从行为和表达中推导，不凭空编造
3. **一致性优先** - 矛盾时以高频出现的特征为准
4. **保留独特性** - 优先提取区别于常人的独特特征

---

## 维度一：核心价值观

### 提取来源
- 价值观描述类数据（最高权重）
- 日记中的感悟和反思
- 对话中表达的立场
- 博客文章中的观点

### 提取规则
1. 从直接表述中提取人生信条（"我认为..."、"我相信..."）
2. 从行为描述中推导价值优先级（花时间最多的 = 最重视的）
3. 从拒绝/否定表述中提取道德底线（"我绝对不会..."）
4. 多次出现的价值观权重 > 单次出现

### 输出格式
```markdown
### 核心价值观
- **[价值1]** - [从数据中推导的具体描述]（支撑：[数据来源]）
- **[价值2]** - [从数据中推导的具体描述]（支撑：[数据来源]）

### 道德底线
- [底线1]（支撑：[数据来源]）

### 价值优先级
1. [最重视的]
2. [次重视的]
...
```

## 维度二：思维模式

### 提取来源
- 思维模式描述类数据
- 决策案例描述
- 问题解决过程记录
- 对话中的推理过程

### 提取规则
1. 从决策案例中提取决策风格（理性分析 vs 直觉判断）
2. 从问题解决描述中提取分析框架
3. 从对话中的推理链提取思维路径
4. 从错误反思中提取思维盲区

### 输出格式
```markdown
### 决策风格
- 类型：[理性/直觉/混合]
- 特征：[具体描述]
- 案例：[从数据中提取的典型案例]

### 分析框架
- [框架描述]

### 问题解决策略
1. [策略1]
2. [策略2]

### 思维盲区
- [盲区描述]
```

## 维度三：表达风格

### 提取来源
- 对话示例（最高权重）
- 社交媒体内容
- 日记的文风
- 博客文章的写作风格

### 提取规则
1. 从对话中提取口语化表达习惯
2. 从多篇内容中统计高频词汇和句式
3. 从不同情绪状态的内容中提取情感表达方式
4. 区分正式场合和非正式场合的表达差异

### 输出格式
```markdown
### 语言风格
- 整体风格：[正式/随意/幽默/严肃/混合]
- 句式偏好：[长句/短句/混合]
- 语气特征：[自信/谦逊/直接/委婉]

### 常用表达
- "[高频表达1]"
- "[高频表达2]"
- "[高频表达3]"

### 情感表达方式
- 开心时：[表达方式]
- 生气时：[表达方式]
- 难过时：[表达方式]
- 认同时：[表达方式]
- 不认同时：[表达方式]
```

## 维度四：知识体系

### 提取来源
- 专业领域内容
- 博客文章主题
- 对话中涉及的话题
- 兴趣爱好描述

### 提取规则
1. 从专业内容中提取领域和深度
2. 从话题分布中提取兴趣广度
3. 从讨论深度中判断知识层次（入门/进阶/专家）
4. 从错误纠正中提取知识盲区

### 输出格式
```markdown
### 专业领域
- **[领域1]** - [深度：入门/进阶/专家]
- **[领域2]** - [深度：入门/进阶/专家]

### 兴趣爱好
- [兴趣1]
- [兴趣2]

### 知识盲区
- [不熟悉的领域]
```

## 维度五：人际关系模式

### 提取来源
- 对话示例（与不同类型的人）
- 社交偏好描述
- 冲突处理案例
- 社交媒体互动风格

### 提取规则
1. 从对话中提取与不同对象的沟通差异
2. 从社交描述中提取偏好（小群体/大群体/独处）
3. 从冲突案例中提取冲突处理策略
4. 从互动频率中判断社交主动性

### 输出格式
```markdown
### 社交偏好
- 群体规模偏好：[小群体/大群体/独处]
- 社交主动性：[主动/被动/看情况]
- 交友标准：[从数据中推导]

### 沟通风格
- 与熟人：[风格]
- 与陌生人：[风格]
- 与上级：[风格]

### 冲突处理
- 策略：[回避/面对/妥协/竞争]
- 案例：[从数据中提取]
```

## 维度六：情感特征

### 提取来源
- 日记中的情感表达
- 对话中的情绪变化
- 压力应对描述
- 情感触发点描述

### 提取规则
1. 从日记中提取情绪基调（整体偏积极/消极/中性）
2. 从多次内容中识别情感触发点
3. 从压力描述中提取应对策略
4. 从情感表达中判断情感外显程度

### 输出格式
```markdown
### 情绪基调
- 整体：[积极/消极/平和/波动]
- 特征：[具体描述]

### 情感触发点
- 正面触发：[什么会让他开心]
- 负面触发：[什么会让他生气/难过]

### 压力应对
- 策略：[具体方法]
- 求助倾向：[是否会寻求帮助]
```

## 维度七：生活态度

### 提取来源
- 工作观描述
- 生活方式描述
- 未来规划
- 消费观、时间分配等

### 提取规则
1. 从工作描述中提取工作观
2. 从生活习惯中提取生活方式偏好
3. 从规划描述中提取未来期望
4. 从消费行为中提取消费观

### 输出格式
```markdown
### 工作观
- 工作意义：[对工作的态度]
- 工作方式：[偏好]
- 职业追求：[目标]

### 生活方式
- 日常节奏：[规律/随意]
- 独处偏好：[是否享受独处]
- 兴趣投入：[如何分配时间]

### 未来期望
- 短期：[1-2年]
- 长期：[5-10年]
```

## 维度八：开场白与示例对话

### 提取来源
- 对话示例中的开场方式
- 自我介绍类数据
- 典型交流场景

### 提取规则
1. 从自述数据中提取开场白素材
2. 从对话示例中选取 2-3 组典型对话作为示例
3. 示例对话应覆盖：轻松闲聊、深度讨论、观点分歧三种场景
4. 开场白应体现人格风格（幽默/严肃/温暖/理性）

### 输出格式
```markdown
### 开场白
[基于人格特征生成的自我介绍，100-200字]

### 示例对话

#### 场景一：轻松闲聊
用户：[消息]
克隆体：[回应]

#### 场景二：深度讨论
用户：[消息]
克隆体：[回应]

#### 场景三：观点分歧
用户：[消息]
克隆体：[回应]
```

## 一致性校验规则

构建完成后，必须进行以下校验：

1. **内部一致性** - 各维度之间不得矛盾
   - 例：声称"重视家庭"但工作观显示"工作至上" → 需要标注或修正
2. **数据一致性** - 特征必须与数据一致
   - 例：提取为"幽默"但数据中无幽默表达 → 需要降级或移除
3. **完整性检查** - 7 个维度必须全部覆盖
   - 某维度数据不足时，标注"数据不足，待补充"

---

## 质量评分

构建完成后，对产物进行质量评分：

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 完整性 | 30% | 7 个维度覆盖度 |
| 真实性 | 30% | 特征与数据一致性 |
| 独特性 | 20% | 区别于通用人格的程度 |
| 可用性 | 20% | 加载后对话自然度 |

总分 100 分：
- 90+：优秀
- 70-89：良好
- 50-69：基本
- <50：建议补充数据后重新构建
