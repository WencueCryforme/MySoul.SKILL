# 产物 SKILL 记忆管理指南

本文档定义产物 SKILL 的记忆管理规范。

---

## 记忆类型

### 1. 短期记忆（对话上下文）
- 当前对话的内容
- 受 AI Agent 上下文窗口限制
- 对话结束后可能丢失

### 2. 长期记忆（持久化）
- 跨对话保留的信息
- 用户偏好、历史交互摘要
- 需要显式存储机制

### 3. 情景记忆（关键事件）
- 与用户的重要交互
- 用户表达的关键观点或情感
- 影响后续回应的参考

---

## 记忆模式

### 有状态模式（默认）

**行为**：
- 每次交互都会累积记忆
- 会记住用户的偏好和历史
- 会根据交互历史调整回应方式

**实现**：
```markdown
## 记忆机制

本 SKILL 为有状态模式。

### 记忆规则
1. 记住用户的姓名和基本信息（如用户主动提供）
2. 记住用户的兴趣偏好
3. 记住之前的对话主题
4. 根据交互历史调整回应风格

### 记忆容量
- 短期记忆：当前对话上下文
- 长期记忆：重要信息摘要

### 记忆重置
用户可请求"重置记忆"清除所有历史。
```

### 无状态模式

**行为**：
- 每次对话独立，不保留历史
- 不会记住用户信息
- 每次都以初始状态开始

**实现**：
```markdown
## 记忆机制

本 SKILL 为无状态模式。

### 记忆规则
1. 不保留任何跨对话信息
2. 每次对话都以初始人格状态开始
3. 不会记住用户的姓名或偏好
```

---

## 记忆管理最佳实践

### 1. 分层存储
- **核心层**：人格特征（不变）
- **交互层**：用户偏好和历史（可变）
- **临时层**：当前对话上下文（自动管理）

### 2. 记忆摘要
- 定期总结对话内容
- 提取关键信息（偏好、观点、情感）
- 压缩冗余信息

### 3. 相关性评分
- 根据时间、频率、重要性排序记忆
- 优先使用高相关性记忆
- 低相关性记忆逐渐淡出

### 4. 用户控制
- 允许用户查看存储的记忆
- 允许用户删除特定记忆
- 允许用户完全重置记忆

## 记忆注入方式

### 方式一：系统提示词注入
将关键记忆写入系统提示词：
```
## 用户信息
- 姓名：[用户姓名]
- 兴趣：[用户兴趣]
- 之前讨论过：[历史话题]
```

### 方式二：对话上下文注入
在对话中自然地引用记忆：
```
你之前提到过对 [话题] 感兴趣，你觉得...？
```

### 方式三：RAG 检索注入
使用向量数据库检索相关记忆：
- 将记忆存储为向量
- 根据当前对话检索相关记忆
- 将检索结果注入上下文

## 记忆一致性

### 问题：记忆与人格冲突
- 例：用户说"我不喜欢 X"，但人格特征显示"喜欢 X"
- 解决：以用户记忆为准，但保持人格一致性

### 问题：记忆过时
- 例：用户之前的偏好已经改变
- 解决：允许用户更新记忆，新记忆覆盖旧记忆

### 问题：记忆冲突
- 例：用户在不同对话中表达矛盾观点
- 解决：以最近的记忆为准，或询问用户确认

## 构建时的记忆配置

在 `/build` 时，可以通过配置指定记忆行为：

```json
{
  "config": {
    "stateful": true,
    "memory": {
      "max_short_term": 10,
      "max_long_term": 100,
      "consolidation_interval": 5,
      "pruning_strategy": "relevance"
    }
  }
}
```

### 配置项说明

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| stateful | 是否有状态 | true |
| max_short_term | 短期记忆最大条数 | 10 |
| max_long_term | 长期记忆最大条数 | 100 |
| consolidation_interval | 记忆整合间隔（对话次数） | 5 |
| pruning_strategy | 记忆修剪策略 | relevance |
